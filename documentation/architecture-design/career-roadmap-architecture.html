MCP protocol: All tool servers expose a JSON-RPC 2.0 interface via stdio or HTTP transport. The agent runtime maintains a tool registry; agents call mcp.call(server_id, tool_name, params) — the runtime resolves, authenticates, rate-limits, and proxies. All calls are permission-scoped and user-revocable.

Key End-to-End Data Flows
1
Roadmap Generation Flow
01User submits goal + uploads CV via Next.js → REST POST
02API Gateway validates JWT, deserialises payload, creates session
03→ Master Orchestrator scores completeness; if ambiguous → Clarification Engine triggers follow-up Qs pushed to client via SSE
04User answers Qs → PATCH /api/session/clarify → Orchestrator resumes
05Task DAG built: CV Agent + Gap Agent + Market Intel Agent run in parallel
06Each agent calls RAG Retrieval Engine for grounding context
07Market Intel Agent calls MCP Job Board + Salary Servers
08Results aggregate → Roadmap Gen Agent synthesises structured roadmap
09Output Validator checks grounding & realism; flags hallucinations
10Roadmap written to PostgreSQL; streamed to client via WebSocket
2
Follow-Up Clarification Flow
01Orchestrator calls Clarification Engine mid-generation
02Engine analyses missing slots: target country? time available? salary goal?
03Generates ≤3 focused questions (not generic); ranked by blocking priority
04Questions pushed to client via SSE event: type: clarification_required
05Frontend renders interactive question UI (not free-text chat)
06User responds → POST /api/clarification/{session_id}
07Answers validated & merged into UserProfile JSON in Redis session
08Orchestrator resumes paused task DAG with enriched context
09If new answers trigger further ambiguity → cycle repeats (max 3 rounds)
10Completeness score ≥ threshold → roadmap generation proceeds
3
Real-Time Market Intelligence Flow
01Market Intel Agent triggered by Orchestrator (or scheduled nightly job)
02Calls MCP Job Board Server → fetches live postings matching user target role
03Calls MCP GitHub Trends Server → trending repos in user's tech stack
04Calls MCP Social Signals Server → HN / Reddit / Twitter for skill signals
05Calls MCP Salary Server → compensation benchmarks by role + country
06Raw data normalised & summarised by LLM (scoped prompt, no hallucination)
07MarketSignal object returned to Orchestrator / Roadmap Gen Agent
08New trends ingested into RAG pipeline → embedding → vector store upsert
09User-relevant alerts written to notifications table in PostgreSQL
10Progress Agent evaluates if plan update is warranted → proposes adaptation
