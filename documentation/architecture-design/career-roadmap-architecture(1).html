Key End-to-End Data Flows
1
Roadmap Generation Flow
01User submits goal + uploads CV → REST POST to FastAPI gateway
02Gateway validates JWT, deserialises payload, creates Redis session
03Master Orchestrator scores completeness → Clarification Engine fires follow-up Qs via SSE if ambiguous
04User answers → PATCH /api/session/clarify → Orchestrator resumes
05Task DAG: CV Agent + Gap Agent + Market Intel Agent run in parallel
06Each agent calls RAG Retrieval Engine for grounded context chunks
07Market Intel Agent calls MCP Job Board + Salary + GitHub servers
08Results aggregate → Roadmap Gen Agent synthesises structured plan
09Output Validator checks grounding & realism; flags hallucinations
10Roadmap written to PostgreSQL; streamed to client via WebSocket
2
Follow-Up Clarification Flow
01Orchestrator invokes Clarification Engine when slot score < threshold
02Engine identifies missing slots: country? weekly hours? salary goal?
03Generates ≤3 focused questions ranked by blocking priority
04Questions pushed to client via SSE: type: clarification_required
05Frontend renders interactive Q UI — not free-text chat
06User responds → POST /api/clarification/{session_id}
07Answers validated & merged into UserProfile JSON in Redis
08Orchestrator resumes paused task DAG with enriched context
09New answers may trigger further clarification (max 3 rounds)
10Completeness score ≥ threshold → generation proceeds
3
Real-Time Market Intelligence Flow
01Market Intel Agent triggered by Orchestrator or nightly Celery job
02Calls MCP Job Board Server → live postings matching user target role
03Calls MCP GitHub Trends Server → trending repos in user's tech stack
04Calls MCP Social Signals Server → HN / Reddit / Twitter signals
05Calls MCP Salary Server → compensation benchmarks by role + country
06Raw data normalised & summarised by LLM (scoped, grounded prompt)
07MarketSignal object returned to Orchestrator / Roadmap Gen Agent
08New trends ingested into RAG pipeline → embed → vector store upsert
09User alerts written to notifications table in PostgreSQL
10Progress Agent evaluates if plan update warranted → proposes adapt.
